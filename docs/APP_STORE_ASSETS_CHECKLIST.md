# App Store Assets Checklist

## Required Screenshots (7 total)

### iPhone Screenshots Required:
1. **6.7" Display (iPhone 14 Pro Max, 15 Pro Max)**: 1290 x 2796 pixels
2. **6.5" Display (iPhone 11 Pro Max, XS Max)**: 1242 x 2688 pixels  
3. **5.5" Display (iPhone 8 Plus)**: 1242 x 2208 pixels

### iPad Screenshots (Optional but recommended):
4. **12.9" iPad Pro**: 2048 x 2732 pixels
5. **11" iPad Pro**: 1668 x 2388 pixels

### App Preview Video (Optional but highly recommended):
6. **6.7" iPhone**: 886 x 1920 pixels, 15-30 seconds, MP4 or MOV
7. **6.5" iPhone**: 886 x 1920 pixels, 15-30 seconds, MP4 or MOV

## Screenshot Content Plan

### Screenshot 1: Home Dashboard
- Show today's timeline with multiple events
- Next nap prediction card visible
- Quick action buttons
- Clean, organized layout

### Screenshot 2: Quick Logging
- Show feed form or quick log in action
- Demonstrate 2-tap logging
- Highlight speed and simplicity

### Screenshot 3: AI Nap Prediction
- Show nap prediction card with time window
- Display baby's age and personalized prediction
- Highlight AI value proposition

### Screenshot 4: History/Analytics
- Show weekly trends or daily summary
- Display charts/graphs (if available)
- Show data visualization

### Screenshot 5: Multi-Caregiver Sync
- Show family sharing or caregiver features
- Display sync status
- Highlight collaboration

### Screenshot 6: AI Assistant
- Show AI chat interface
- Display helpful response
- Highlight 24/7 support

### Screenshot 7: Settings/Privacy
- Show privacy controls
- Display data export options
- Highlight trust and security

## App Preview Video Script (30 seconds)

**0-5s**: Quick montage of logging (feed, diaper, sleep) - "Log in 2 taps"
**5-10s**: Show nap prediction appearing - "AI predicts next nap"
**10-15s**: Show partner sync/notification - "Syncs with partner instantly"
**15-20s**: Show AI assistant answering question - "AI answers your questions"
**20-25s**: Show analytics/insights - "See patterns and trends"
**25-30s**: App icon + tagline - "Nuzzle - Track smarter, not harder"

## Tools for Generating Screenshots

1. **Xcode Simulator**:
   - Run app in simulator
   - Use Cmd+S to capture screenshot
   - Or use `xcrun simctl io booted screenshot screenshot.png`

2. **Fastlane Frameit** (Recommended):
   ```bash
   fastlane frameit
   ```
   - Adds device frames automatically
   - Professional presentation

3. **Manual Tools**:
   - Sketch, Figma, or Photoshop
   - Use device frame templates
   - Ensure text is readable at small sizes

## Testing Screenshots

- View on actual device at App Store size
- Ensure text is readable without zooming
- Check contrast and visibility
- Verify no sensitive data is visible
- Test in light and dark mode (if showing dark mode)

## Submission Checklist

- [ ] 7 screenshots generated for required sizes
- [ ] All screenshots show actual app functionality
- [ ] No placeholder text or "lorem ipsum"
- [ ] No sensitive user data visible
- [ ] App preview video created (optional but recommended)
- [ ] All assets meet Apple's guidelines
- [ ] Text overlays are readable at small sizes
- [ ] Screenshots demonstrate key value propositions

## Notes

- Screenshots are the first thing users see
- Show real functionality, not mockups
- Highlight unique features (AI, sync, speed)
- Use actual baby data (anonymized) for realism
- Consider A/B testing different screenshots
